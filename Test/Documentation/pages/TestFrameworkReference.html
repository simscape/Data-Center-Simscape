Test Framework Reference

Detailed API reference for all framework classes, functions, and utilities

TestExecutor

Test/Framework/TestExecutor.m

Central orchestrator that coordinates test discovery, execution, and reporting. Reads a configuration struct from any testConfig.m file, discovers targets via TestFileDiscovery, builds parameterized suites, configures the runner with plugins, and executes tests.

Static Methods

MethodDescription
runAll(config) Discovers all test targets (models, workflows, custom tests), bridges config.SuppressedWarnings to the smoke test TestCase, runs the full suite with code coverage, and generates all configured reports. Returns matlab.unittest.TestResult array.
runModelsOnly(config) Runs only model simulation tests (TestModelSimulation).
runWorkflowsOnly(config) Runs only workflow script tests (TestScriptExecution).
runCustomOnly(config) Runs only custom TestCase files and/or directories specified in config.CustomTests.
getCodeAnalyzerPaths() Returns output paths (.mat and .sarif) for Code Analyzer results.
dryRun(config) Previews which files would be tested without executing any tests.

Syntax

results = TestExecutor.runAll(config);
results = TestExecutor.runModelsOnly(config);
results = TestExecutor.runWorkflowsOnly(config);
results = TestExecutor.runCustomOnly(config);
paths   = TestExecutor.getCodeAnalyzerPaths();
TestExecutor.dryRun(config);

Input

config
Configuration struct returned by any testConfig() function. Contains fields: Models, Workflows, CodeCoverage, Reports, CustomTests, SuppressedWarnings.

ProjectSmokeTests

Test/Framework/ProjectSmokeTests.m

Parameterized matlab.unittest.TestCase class that provides two smoke test methods. Parameters are injected externally by TestExecutor using Parameter.fromData.

TestClassSetup Methods

MethodDescription
setupWorkingFolder Applies WorkingFolderFixture so each test runs in an isolated temporary directory.
suppressConfiguredWarnings Reads warning IDs from config.SuppressedWarnings (via appdata bridge) and applies SuppressedWarningsFixture for all listed IDs.

Test Methods

MethodDescription
TestModelSimulation(modelName) Loads the specified Simulink model via load_system() and simulates it via sim(). Passes if simulation completes without error.
TestScriptExecution(scriptName) Executes the specified script in the base workspace inside verifyWarningFree(). Passes if the script completes without warnings or errors.

Test Parameters

modelName
Cell array of model file names (e.g., {'model1.slx', 'model2.slx'}). Populated externally.
scriptName
Cell array of script file names (e.g., {'workflow1.mlx'}). Populated externally.

Note: The default value {"__skip__"} is used as a sentinel. Tests use assumeNotEqual to gracefully skip when no targets are parameterized.

TestFileDiscovery

Test/Framework/TestFileDiscovery.m

File discovery engine using a two-phase inclusion/exclusion approach. Called by TestExecutor to locate models, scripts, and coverage source files.

Static Methods

MethodDescription
find(Name,Value) Discovers files matching the given search patterns, file types, and filters. Returns a deduplicated, filtered string array of absolute file paths.

Discovery Pipeline

  1. Phase 1 (Inclusion):
    • Searches each directory in IncludeDirectories recursively for files matching FileTypes.
    • Adds each entry from IncludeFiles (resolved relative to project root).
    • Deduplicates the combined list via unique().
  2. Phase 2 (Exclusion):
    • ExcludeFunctionFiles: Removes .m files that define functions.
    • ExcludeDirectories: Removes files under specified directories.
    • ExcludeFiles: Removes files whose relative path matches the exclude list.

Warning: Exclusion always wins over inclusion. A file present in both IncludeFiles and ExcludeFiles will be excluded from the final list.

Name-Value Arguments

NameTypeDescription
ProjectRootstringAbsolute path to the project root. (Required)
IncludeDirectoriesstring arrayDirectories relative to the project root to search. Always searched recursively. (Required)
FileTypesstring arrayFile type labels: "M-file", "SLX-file", "MDL-file", "MLX-file". (Required)
IncludeFilesstring arraySpecific file paths (relative) to merge into candidate list. Default: string.empty
ExcludeDirectoriesstring arrayDirectory paths relative to project root to exclude. Default: string.empty
ExcludeFilesstring arrayFile paths relative to project root to exclude. Default: string.empty
ExcludeFunctionFileslogicalExclude .m function files. Default: false. Set automatically by the framework.
DisplayListlogicalPrint discovered files to command window. Default: false

TestReportBuilder

Test/Framework/TestReportBuilder.m

Factory class that creates and configures test runner plugins for report generation.

Static Methods

MethodDescription
addJUnitPlugin(outputFolder, releaseStr) Creates an XMLPlugin producing JUnit format.
addHTMLReportPlugin(outputFolder, releaseStr) Creates a TestReportPlugin producing interactive HTML.
addCoveragePlugin(fileList, outputFolder, releaseStr) Creates a CodeCoveragePlugin measuring coverage on the given source files.
getCodeIssuesPath(outputFolder, releaseStr) Returns output paths for Code Analyzer results: [.mat, .sarif]
ensureFolder(folderPath) Creates the output folder if it does not exist.

excludeByDirectory

Test/Framework/Utilities/excludeByDirectory.m

Removes file paths that reside under specified directories (relative to project root). Used during Phase 2 (Exclusion) of the discovery pipeline.

Syntax

filtered = excludeByDirectory(paths, directories, projectRoot);

Inputs

InputTypeDescription
pathsstring arrayAbsolute file paths to filter.
directoriesstring arrayDirectory paths relative to project root to exclude.
projectRootstringAbsolute path to the project root.

Note: Each directory entry is resolved to fullfile(projectRoot, dir). A file is excluded if its path starts with the resolved directory path.

isFunctionFile

Test/Framework/Utilities/isFunctionFile.m

Determines if a .m file defines a function by reading the first non-comment, non-blank line. Returns true for functions, false for scripts, classdefs, or unreadable files.

Syntax

isFunc = isFunctionFile(file);

testConfig Files

Test/testConfig_<1>.m   Test/testConfig_<2>.m

Configuration functions that return a struct defining a self-contained test subset. Each file defines what to test, what to ignore, and where to write reports.

See Configuration Guide for full details.
