Getting Started

Set up and run your first tests

Prerequisites

  • MATLAB R2023b or later (R2024b+ recommended for full feature support)
  • Simulink (for model simulation tests)
  • The Test/Framework folder (and subfolders) added to the MATLAB path

Note: The framework uses matlab.unittest, CodeCoveragePlugin, XMLPlugin, and TestReportPlugin, all of which are included with MATLAB (no additional toolboxes required for core functionality).

Setup

  1. Copy the entire Test/ folder into your project root directory.
  2. Add the Test/Framework folder (and subfolders) to the MATLAB path. This can be done via a MATLAB Project, addpath, or your startup script.
  3. Verify by running: which TestExecutor — it should return the path to Test/Framework/TestExecutor.m.

Quick Start

The fastest way to verify the framework is working:

% Load a test configuration
config = testConfig();

% Preview what will be tested (no tests are run)
TestExecutor.dryRun(config);

Dry Run

The dry run previews which files the framework will discover and test, without actually executing any simulations or scripts. This is useful for validating your configuration before committing to a full test run. 

% Dry run shows discovered models, workflows, custom tests, and coverage files
TestExecutor.dryRun(testConfig());

% Example output:
% ===== DRY RUN: Test File Discovery =====
%
% --- Models (TestModelSimulation) ---
% [TestFileDiscovery] Found 3 file(s):
%   C:\project\Components\HarnessModels\model1.slx
%   ...
%
% ===== Summary =====
%   Models:              3
%   Workflows:           2
%   Custom Tests:        0
%   Coverage:            15 source files
%   Total test points:   5
% ==================================

Running Tests

Once you are satisfied with the dry run output, execute the full test suite:

Run All Tests

config = testConfig();
results = TestExecutor.runAll(config);

Run Only Models

config = testConfig();
results = TestExecutor.runModelsOnly(config);

Run Only Workflows

config = testConfig();
results = TestExecutor.runWorkflowsOnly(config);

Run Only Custom Tests

config = testConfig();
results = TestExecutor.runCustomOnly(config);

Tip: Set config.Reports.OpenInBrowser = true for interactive use. The coverage report will open automatically in your default browser after the test run completes.

Viewing Reports

After running tests, reports are generated in:

Test/ProjectTestReport/<FolderName>/
├── TestResults_<version>.xml          % JUnit XML (for CI)
├── TestReport_<version>.html          % Interactive HTML report
└── CodeCoverage_<version>/            % Code coverage HTML report
    └── CoverageReport_<version>.html

Open the HTML files in any browser to review results interactively.

Framework Assumptions

The test framework makes the following assumptions about your project setup. Ensure these are met before running tests.

Project Structure

  • config.ProjectRoot must point to a valid directory. The framework does not require a MATLAB Project (.prj file) — any directory structure works.
  • The Test/ folder is located directly at the project root level (i.e., <ProjectRoot>/Test/).
  • The internal folder structure Test/Framework/ must remain intact. The framework expects ProjectSmokeTests.m at Test/Framework/ProjectSmokeTests.m.
  • Report output is written to Test/ProjectTestReport/<FolderName>/, which is auto-created if it does not exist.

MATLAB Path

  • All models referenced in the configuration must be on the MATLAB path. The framework calls load_system(modelName) by file name only (no full path).
  • All scripts and workflows referenced in the configuration must be on the MATLAB path or resolvable by run().
  • The Test/Framework folder and its subfolders must be on the MATLAB path so that TestExecutor, ProjectSmokeTests, and utility functions are accessible.
  • Custom test files (CustomTests.Files) and directories (CustomTests.Directories) are resolved relative to config.ProjectRoot, not the current working directory.

Test Execution

  • Model simulation tests call sim(modelName) with no additional parameters. Models must have valid solver settings and all required parameters in the base workspace or loaded via callbacks.
  • Workflow script tests execute scripts inside verifyWarningFree(). Any warning issued during script execution will cause the test to fail.
  • Scripts are executed in the base workspace via evalin('base', ...). Variables created by scripts persist in the base workspace.
  • Tests run inside a temporary working folder (via WorkingFolderFixture). Scripts that rely on the current directory being the project root may fail.
  • All Simulink models and figures opened during a test are automatically closed in teardown. Do not rely on models remaining open between tests.
  • The MATLAB:hg:AutoSoftwareOpenGL graphics warning is suppressed by default during test execution.

Configuration

  • config.ProjectRoot is required and must be an absolute path to an existing directory.
  • All other paths in testConfig fields are relative to config.ProjectRoot.
  • IncludeDirectories must be a string array. Each entry is a directory relative to the project root that is searched recursively.
  • FileTypes must use the exact labels: "M-file", "SLX-file", "MDL-file", "MLX-file".
  • Reports.FolderName must be non-empty and must not contain characters invalid in file system paths.
  • Custom test files must be matlab.unittest.TestCase subclasses.

Report Generation

  • Reports are named using the MATLAB release string (e.g., TestResults_R2025b.xml).
  • If config.Reports.OpenInBrowser = true, the coverage report is opened using MATLAB's web() function.
  • Code coverage is only generated if GenerateCodeCoverage = true and at least one coverage source file is discovered.
