The custom block is based on simplified physics for the electrochemical reactions happening inside a Vanadium Redox Flow Battery, VRFB. The block is available both in thermal and non-thermal specification.

Mathematical Model

The electrochemical reaction in a Vanadium Redox Flow Battery has four different species of vanadium. The electrochemical reactions, at the positive and the negative electrode, are given below.

In this document, the concentration for species are denoted by respectively. Suffix of 'cell' or 'tank' is used to indicate the Va specie in the cell stack or the electrolyte tank. The electrodes are separated by a thin polymeric membrane. The polymeric membrane allows the hydrogen ions to pass while preventing any electron migration. The electrons flow through the external circuit, forming the current from the battery. Multiple cells are connected in series and parallel to achieve the desired energy and power. The cell stack is fed by electrolyte in tanks and the entire assembly, along with pumps, is housed in a rectangular enclosure. The fan at top of the enclosure helps cool the battery stack. The battery electrical power depends on the cell current i, the electrolyte flow rate Q, and the Va specie concentration . To model the VRFB dynamics, the material balance equations for all the Va species coupled with a thermal and hydraulics model must be solved. The thermal model is an optional feature and the custom block runs for an isothermal case too.

Flow Model

The hydraulics of the battery are modeled by capturing the pressure drop due to coolant flow. There is pressure drop due to flow through stacks and flow through pips connecting the tank and the stacks. The stack pressure drop is modeled as a look-up-table where test data on stack pressure drop at different flow rates must be provided. The pressure drop through pipes connecting the tank and stack is computed using a Simscape internal function (foundation.thermal_liquid.elements.pipe_friction).

Experimental values of pressure drop across the stack for all the specified break-points for flow-rate are required for this model. The pressure drop due to piping from tank-to-stack and stack-to-tank is computed using the pipe_friction method in the Simscape Foundation library. In the custom component, the length of the pipe used in the pipe_friction method is based on the user specified parameter of pipe length and the calculated series-stack length.

Thermal Model

The cell stack is housed in a container. The heat from the cell stack is dissipated to the air inside the container where the fans help in cooling. The flowing electrolyte enables energy exchange between tank electrolyte and the cell stack electrolyte. The flowing electrolyte also dissipates heat through the pipe walls to the air inside the container. The container has thermal mass and it acts as a barrier to heat transfer from the inside air to the ambient. Thermal model is an optional feature that can be enabled using parameter Thermal Model on the library block mask.

Block Parameters

The VRFB is specified using one or more stacks. Each stack is a collection of series connected cells. All properties, break-points, for the battery are specified at a stack level, be it the equivalent circuit model parameters, the pressure drop across stack, or the thermal model. You can configure multiple stacks in series and parallel to achieve the desired power and energy ratings. The picture below shows the typical arrangement of stacks implements in the custom component Vanadium Redox Flow Battery.
The block has one input port for specifying tank to each series-stack electrolyte flow-rate, two electrical nodes, and one (optional) thermal node. You must specify breakpoints for electrolyte flow-rate and (optional) temperatures values and the measured ohmic resistance as well as the RC pair value for the battery equivalent circuit.

Break-points

  • Stack flowrate vector, flowrateLUT, specified as a vector of flow-rate break-point values.
  • Thermal model, thermalManagement, specified as a drop-down list. You can select from Enabled or Disabled. If you select Enabled, then you must specify additional parameters in the Thermal section.
  • Temperature vector, temperatureLUT, specified as a vector of temperature break-point values. This is required when you select the Thermal Model as Enabled.
  • Initial cell state of charge, SOC, specified as a scalar value. This specifies the initial battery state-of-charge value.
  • Initial temperature, initialTsys, specified as a scalar value. This specifies the initial battery temperature value.

Electrolyte Properties

  • Electrolyte density, electrolyteDen, specified as a scalar value.
  • Electrolyte heat capacity, electrolyteCp, specified as a scalar value.
  • Electrolyte kinematic viscosity, electrolyteKinVis, specified as a scalar value.
  • Cell Vanadium concentration check, msgWarn, specified as a drop-down list. If you select none, then no warning or errors are displayed in case of vanadium concentration crossing the limits set by Vanadium molarity in electrolyte. If you select warning, then a warning message is displayed, and if you select error, then an error is thrown and the simulation stops.
  • Cross diffusion Model, crossDiffModel, specified as a drop-down list. You can select from Disabled or Enabled.
If you select Cross diffusion Model as Enabled, then you must specify the additional following parameters.
  • Membrane thickness, thMemb, specified as a scalar value.
  • Va(II) diffusion coefficient through membrane, diffCoeff2, specified as a scalar value. This defines the diffusion constant for the vanadium specie .
  • Va(III) diffusion coefficient through membrane, diffCoeff3, specified as a scalar value. This defines the diffusion constant for the vanadium specie .
  • Va(IV) diffusion coefficient through membrane, diffCoeff4, specified as a scalar value. This defines the diffusion constant for the vanadium specie .
  • Va(V) diffusion coefficient through membrane, diffCoeff5, specified as a scalar value. This defines the diffusion constant for the vanadium specie .
  • Activation energy for diffusion reaction, activationE, specified as a scalar value. This defines the activation energy for the Arrhenius dependence of diffusion coefficient.
  • Shunt current model, shuntResModel, specified as a drop-down list. You can select from Disabled, Equation, and User defined options. When you select Disabled, the cross diffusion model is not activated. When you select Equation, the shunt resistance value is calculated inside the function vanadiumRedoxFlowShuntRes based on the parameter Electrolyte conductivity, electrolyteK and internally computed values of shunt series and parallel resistances. If you select the User defined option, you must specify the Effective self discharge resistance value, shuntRes, for the battery pack.
  • Minimum state of charge, minStateOfCharge, specified as a scalar value. This specifies the minimum value of battery state-of-charge below which the Cell vanadium concentration check, msgWarn, option will either display an error, or a warning, based on the option selected.
  • Maximum state of charge, maxStateOfCharge, specified as a scalar value. This specifies the maximum value of battery state-of-charge beyond which the Cell vanadium concentration check, msgWarn, option will either display an error, or a warning, based on the option selected.

Stack Electrical

If the Thermal Model is Enabled, all the parameters in this section must be specified as a matrix of Simscape values. The matrix row and column sizes must be equal to the length of the Temperature vector and the Stack flowrate vector, respectively. If the Thermal Model is Disabled, all the parameters in this section must be specified as a vector of Simscape values, with the length of vector being equal to the Stack flowrate vector.
  • Vanadium molarity in electrolyte, molar, specified as a scalar value.
  • Single cell electrode potential, V0, specified as a scalar value. This specifies the voltage difference between the two electrodes in the cell.
  • Stack discharge ohmic resistance, LUTdischgR0, specified as a vector or matrix values based on Thermal Model selection. This parameter specifies ohmic resistance value based on discharge test conditions.
  • Stack discharge dynamic resistance, LUTdischgRes, specified as a vector or matrix values based on Thermal Model selection. This parameter specifies dynamic resistance value based on discharge test conditions.
  • Stack discharge time constant, LUTdischgTau, specified as a vector or matrix values based on Thermal Model selection. This parameter specifies time-constant based on discharge test conditions.
  • Stack charge ohmic resistance, LUTchgR0, specified as a vector or matrix values based on Thermal Model selection. This parameter specifies ohmic resistance value based on charge test conditions.
  • Stack charge dynamic resistance, LUTchgRes, specified as a vector or matrix values based on Thermal Model selection. This parameter specifies dynamic resistance value based on charge test conditions.
  • Stack charge time constant, LUTchgTau, specified as a vector or matrix values based on Thermal Model selection. This parameter specifies time-constant based on charge test conditions.
  • Stack length, stackLen, specified as a scalar value.
  • Stack width, stackWid, specified as a scalar value.
  • Stack height, stackHgt, specified as a scalar value.
  • Stack thermal mass, stackMassCp, specified as a scalar value.

Stack Flow

  • Stack pressure drop at different flowrate points, stackPrDropLUT, specified as a vector of scalar values. The length of the vector must be equal to the length of vector in Stack flowrate vector. This vector denotes pressure drop inside a battery stack at different flow-rates.
  • Stack effective pipe length, pipeLength, specified as a scalar value. This denotes the effective pipe length till the battery stacks.
  • Stack pipe hydraulic diameter, pipeDia, specified as a scalar value. This denotes the effective diameter of the pipes to the battery stacks.
  • Stack pipe cross-sectional area, pipeArea, specified as a scalar value. This denotes the cross-sectional area of pipes till the battery stack.
  • Stack pipe roughness, pipeRoughness, specified as a scalar value.

Battery Stacks

  • Number of cells per stack, Nc, specified as a scalar value.
  • Number of series connected stacks, Ns, specified as a scalar value.
  • Number of parallel units of series connected stacks, Np, specified as a scalar value.
  • Series stack flow maldistribution, seriesFlowMalDistr, specified as a vector of scalar values. The length of the vector is equal to the number of series connected stacks in the battery. The sum of all values in this vector must equal the Number of stacks. This vector is used to denote flow maldistribtuion of the flow distribution to series connected stacks.

Container Thermal

  • Container volume, containerVolume, specified as a scalar value.
  • Container surface area for ambient heat loss, containerArea, specified as a scalar value. This specifies the surface area of the container exposed to ambient.
  • Container thermal mass, containerThermalmass, specified as a scalar value.
  • Number of cooling fans, nFans, specified as a scalar value.
  • Cooling fan flow rating, fanCapacity, specified as a scalar value.
  • Heat transfer coefficient from ambient to container walls, extHTC, specified as a scalar value.
