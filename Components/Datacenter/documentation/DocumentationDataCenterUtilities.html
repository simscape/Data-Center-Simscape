Utilities for Automated Build

getDataCenterNamePlateRatingTable

This function is used to modify parameters for server required by the data center build utilities like buildDatacenter and createDatacenterWithEnclosure. The function outputs a struct with two tables, one of size (6,2) and the other of size (5,1). The tables contain parameters for data center server like Server Idle Power, Power Supply Efficiency, Server Actual To Name Plate Ratio, Server Mass & Specific Heat, Peak Power of CPU, Memory, Disk, PCI, Motherboard, and Fans.
Example:
data = getDataCenterNamePlateRatingTable(...
Mass=simscape.Value(30,"kg"),...
PeakPowerMotherboard=simscape.Value(34,"W"),...
IdlePower=simscape.Value(56,"W"),...)
Display=true);

createDataCenterWithEnclosure

This function executes buildDatacenter and buildEnclosureForDatacenter. The name value pair for this function is a combined list of name value pair for these two functions. This utility also lets you save the model files generated at your specified location (name-value-pair of SaveAtFullPath) and also specify the type of file you want to save it as (name-value-pair of SaveModelAs).
Example:
data = getDataCenterNamePlateRatingTable(...
Mass=simscape.Value(30,"kg"),...
PeakPowerMotherboard=simscape.Value(34,"W"),...
IdlePower=simscape.Value(56,"W"),...)
Display=true);
 
createDataCenterWithEnclosure(NumPDUunitsX=3,NumPDUunitsY=4,...
ModelName="ModelDatacenterLiquidCooling",...
ModelOption="Thermal",...
ModelEnclosure=false,...
RatingDatacenter=simscape.Value(6000,"kW"),...
ServerNamePlateRating=data.nameplateRating,...
ServerPowerSystemSpec=data.powerSystemSpecs,...
ThermalRes=simscape.Value(1e-6,"K/W"),...
Diagnostics=true);
Additionally, you may find the block Utilization Converter custom library block useful in converting a 2D matrix of server utilization data (based on NumPDUunitsX and NumPDUunitsY values required by createDatacenterWithEnclosure) into a vector form required by the large data center created above.

getCompositeDataCenterRating

The utilties in this project help create a large data center with multiple Datacenter Server Unit custom library block. Use this function to determine or confirm the rating of (composite) data center created using such utilities.
Arguments:
  • BlockPath: Specified as a string and denotes the path to the composite data center model.
  • Display: Specified as a logical value to enable or disable display of result.
Example:
% If in a SLX model DataCenterCooling, you have a subsystem called 'Datacenter',
% that contains the 'Datacenter Server Unit' library blocks, then specify the
% path as shown below.
getCompositeDataCenterRating(BlockPath="DataCenterCooling/Datacenter",...
Display=true);

Other Useful Utilities

buildDataCenter

This function is used to connect multiple data center servers in a certain topology. There a 3 different variants available, thermal, electrical, and electrothermal.
Parameters:
  • NumPDUunitsX: Number of power distribution units (PDU) of data center in X direction.
  • NumPDUunitsY: Number of power distribution units (PDU) of data center in Y direction.
  • ModelName: Name of Simscape model.
  • ModelOption: Model fidelity; 3 different options available - thermal, electrical, and electrothermal.
  • RatingDatacenter: A simscape value to specify desired power rating of the data center.
  • ServerNamePlateRating: A table of size (6,2) to specify server name plate rating parameters. Default values can be edited using function getDataCenterNamePlateRatingTable.
  • ServerPowerSystemSpec: A table of size (5,1) to specify data center parameters. Default values can be edited using function getDataCenterNamePlateRatingTable.
  • PowerSupplyEff: A scalar to specify the percentage efficiency of power supply. This is an optional argument required for Electrical and Electrothermal model fidelities.
  • ThermalRes: A scalar to specify the thermal resistance of the data center stack bottom rack and the ground.
  • Diagnostics: A logical, when set to true, outputs key updates from the function/utility.
Example:
data = getDataCenterNamePlateRatingTable(...
Mass=simscape.Value(30,"kg"),...
PeakPowerMotherboard=simscape.Value(34,"W"),...
IdlePower=simscape.Value(56,"W"),...)
Display=true);
 
datacenter = buildDataCenter(NumPDUunitsX=5,...
NumPDUunitsY=6,...
ModelName="MyModel",...
ModelOption="Electrothermal",...
RatingDatacenter=simscape.Value(50,"kW"),...
ServerNamePlateRating=data.nameplateRating,...
ServerPowerSystemSpec=data.powerSystemSpecs,...
ThermalRes=simscape.Value(5,"K/W"),...
Diagnostics=true);

buildEnclosureForDataCenter

This function helps build a rectangular enclosure or room around the data center.
Arguments:
  • ModelName: Specified as a string and denotes the name for the Simscape model.
  • EnclosureName: Specified as a string and denotes the name for the room/enclosure subsystem.
  • DatacenterModel: A struct specifying data center properties.
  • Diagnostics: When set to true, outputs key updates from the build process.
  • RoomLength, RoomWidth, RoomHeight: Simscape values for room/enclosure length, width, and height.
  • WallThickness: Simscape value to specify the wall thickness for the room/enclosure.
  • RoomRotate: Simscape value for the value of which the room is to be rotated.
  • PercentAirVolInRoom: Percent volume of room that is empty or has stagnant air.
Example:
roomModel = buildEnclosureForDataCenter(ModelName=NameValueArgs.ModelName,...
DatacenterModel=datacenter,...
EnclosureName=NameValueArgs.EnclosureName,...
PercentAirVolInRoom=8,...
RoomLength=simscape.Value(4,"m"),...
RoomWidth=simscape.Value(3.5,"m"),...
RoomHeight=simscape.Value(2.5,"m"),...
WallThickness=simscape.Value(1e-1,"m"),...
RoomRotate=simscape.Value(15,"deg"),...
Diagnostics=false);
